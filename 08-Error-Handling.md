## 2. Exceptions (try / catch / finally)

Exceptions are used to handle **unexpected situations** in a controlled way.

```php
try {
    if (!file_exists("file.txt")) {
        throw new Exception("File not found");
    }
} catch (Exception $e) {
    echo $e->getMessage();
} finally {
    // cleanup or logging
}
```

### Keywords:

- **try** â†’ Code that may fail
- **throw** â†’ Raise an exception
- **catch** â†’ Handle the exception
- **finally** â†’ Always runs

## 3. Errors vs Exceptions (Important)

|Errors|Exceptions|
|---|---|
|Generated by PHP engine|Thrown by code|
|Warnings, notices, fatal errors|Exception objects|
|Not always catchable|Catchable|

### PHP 7+
Most fatal errors are thrown as **Error objects**.

```php
try {
    riskyCode();
} catch (Throwable $e) {
    echo $e->getMessage();
}
```

## 4. Custom Exceptions

Used to give **meaningful error names**.

```php

throw new InvalidAmountException("Amount must be positive");
```
